# `.console/` Reconciliation — design

> **Status:** DESIGN (for approval). Supersedes the first-draft central table.
> Adversarially refined; see §5 for the critiques that shaped it.
> The implementation contract derived from this design is
> [console-reconciliation-spec.md](./console-reconciliation-spec.md).

## 1. Problem

Every repo's `.console/log.md` / `backlog.md` grows without bound, mixing
(a) active work, (b) completed work whose *design* may or may not have reached
durable docs, and (c) operator-private narration that — in public repos — leaks
private-repo names into tracked, GitHub-visible files (B1 misses it because
`.console/**` is default-excluded). The Phase-4 hot-trim bounds the *compiled*
blob but leaves the source files untouched.

We want to **prune the source files** — but never lose completed-work design,
and never relocate the leak. Three invariants:

- **I1 — Consolidate before prune:** a `done` item's history is archive-eligible
  only once its design exists in durable docs (verifiable by tooling).
- **I2 — Boundary:** private-repo names never sit in public tracked files; pruned
  private-laden history moves to where private names are *allowed*.
- **I3 — Nothing destroyed:** prune = move (to archive) + trim, never delete.

## 2. Architecture — three layers

### Layer A — Recurring gate (always-on, cheap; Custodian)

Two detectors keep the problem from re-accumulating silently:

- **R1 (advisory):** a `.console/` source file exceeds its line budget →
  "reconciliation due." Cheap signal, no judgement.
- **R2 (fail-closed):** a **scrub-target private name** (the private-repo
  identifiers sourced from the boundary artifact — not enumerated here, per I2)
  appears in a **public** repo's tracked
  `.console/`. This is the original leak; the fix is to **stop default-excluding
  `.console/**` for these names** in B1 (or ship R2 alongside). `PrivateManifest`
  is *not* a scrub target — see [[private-manifest-generalization]].

### Layer B — Reconciliation pass (on-demand; `cl reconcile`)

A per-repo **worksheet** drives a tool. The worksheet is **untracked**
(`.console/reconcile.yaml`, gitignored) — it's scaffolding, not a second backlog,
and being untracked it can name owners freely without leak risk.

```yaml
# .console/reconcile.yaml  (untracked worksheet)
schema: 1
repo: Custodian
items:
  - id: detectors-t6-t8
    title: "T6/T7/T8 test-presence trio"
    status: done            # done | partial | incomplete
    owner: Custodian        # owning repo; owner != repo ⇒ route, don't archive here
    doc: []                 # durable design locations; empty + done ⇒ DOC GAP
  - id: refactor-phases-0-15
    title: "Refactor master phase list"
    status: done
    owner: Custodian
    doc: [docs/design/detector_disposition_matrix.md]
  - id: cross-repo-debt-oc
    title: "Audit debt F3/F1/C29"
    status: incomplete
    owner: OperationsCenter # ⇒ surfaces in OC's backlog, never pruned from here
```

Commands:

- **`cl reconcile check`** — the I1 gate. Fails if any `status: done` item has an
  empty or non-existent `doc[]`; lists DOC GAPs; lists cross-repo items
  (`owner != repo`) for routing; rejects scrub-target names in any field.
- **`cl reconcile prune`** — runs only when `check` is green. For each `done`
  item with `owner == repo`: moves its history out of source → archive (Layer C),
  trims `log.md`/`backlog.md` to *active + recent-N*, writes a pointer + a scrubbed
  one-line `CHANGELOG.md` entry (the public-safe record of what was done).
  Cross-repo `incomplete` items are emitted as routing suggestions for the owner
  repo's backlog.

### Layer C — Archive destination: the private side

Pruned history → **`<private-manifest-repo>/archive/console/<repo>/{log,backlog}-<cutoff>.md`**.

This is sanctioned by the existing visibility model — *"the private-manifest repo is private
(superset — can host state involving public repos; PM cannot host state involving
private repos)."* Two payoffs: (1) public repos shed their raw operator logs;
(2) any private names embedded in that completed history land in the **private**
repo where they're allowed — so the leak is *resolved*, not relocated. The public
repo retains a pointer + the scrubbed CHANGELOG summary, so public contributors
still see *what* shipped, just not the raw log.

### Central index (generated dashboard)

`PlatformManifest/docs/architecture/console-reconciliation-status.md` — generated
locally over all clones: per **public** repo, counts (done/partial/incomplete),
doc-gaps, prune-ready?. Private repos appear as a single aggregate count, never
itemized. Hand-edited nothing; regenerated by `cl reconcile index`.

## 3. Boundary handling (I2), concretely

- Scrub targets (the boundary-artifact private identifiers) → genericize in any content
  that *stays* tracked-public ("a private downstream repo"); content that leaves
  to the private archive needs no scrubbing (private names allowed there).
- `PrivateManifest` is out of scope here (separate architecture item:
  generalize to a discovered private-manifest *role*).
- The worksheet and per-repo archive for **private** repos stay inside those
  repos / the private side; never in this public doc.

## 4. Why not simpler / why not bigger

- **Not just gitignored `.console/archive/` in-repo:** that keeps private-laden
  history on the public local tree and in nobody's backup discipline; the private
  side is the correct home (I2).
- **Not a second live backlog:** the worksheet is transient per-pass scaffolding;
  durable truth stays = docs (design) + archive (history) + CHANGELOG (summary).
- **Not over-built:** Layer A is two cheap detectors that run in CI forever;
  Layers B/C run on demand when A fires. We don't maintain per-item YAML for all
  22 repos in perpetuity.

## 5. Adversarial critique → refinements applied

1. **"Coupling public repos to the private-manifest repo hides history from contributors."**
   → Mitigated: git history is untouched; only operator `.console/` narration
   moves, and a scrubbed CHANGELOG + pointer stays public. The moved content was
   leaking private names anyway — it *shouldn't* be public.
2. **"reconcile.yaml duplicates backlog.md → drift."** → Made the worksheet
   **untracked, transient** scaffolding, not a competing tracker. Durable truth is
   docs+archive+CHANGELOG.
3. **"One-shot YAML rots; who maintains it?"** → Split into a cheap *recurring*
   gate (Layer A, always-on) and an *on-demand* pass (Layer B). Nothing permanent
   to rot.
4. **"Over-engineering a cleanup."** → It isn't one-time: `.console/` grows
   continuously, so a recurring bound + a documented-before-archive gate earns its
   keep. Heavy parts are on-demand only.
5. **"Central index could itself be a leak."** → Public repos only, private repos
   as an opaque count, generated (no hand-leak), owner fields genericized by the
   tool.
6. **"done/partial/incomplete too coarse for the gate."** → Gate applies only to
   `done` (archive-eligible); partial/incomplete always stay active. Coarse is
   sufficient.

## 6. Build order

1. Layer A R2 detector + B1 `.console/` scrub-target tightening (closes the live
   leak immediately) — smallest, highest-value.
2. `cl reconcile check` + the worksheet schema.
3. Custodian pilot: author its worksheet, fill the 7 DOC GAPs (T6/T7/T8, X3,
   CV1–CV3) as catalog entries **and** full usage docs, `check` green.
4. `cl reconcile prune` + Layer C archive into the private-manifest repo; pilot-prune
   Custodian; recompile; verify size + zero public leak.
5. `cl reconcile index`; fan out to remaining public repos; private repos
   in-place; OperationsCenter in a paused-loop window.
